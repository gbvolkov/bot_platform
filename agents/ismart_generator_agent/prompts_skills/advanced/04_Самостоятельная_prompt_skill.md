# Самостоятельная: промпт и skill генерации самостоятельной работы

## Назначение
самостоятельная работа и самоконтроль

## Когда использовать
Используй для занятий, где `lesson.content_flags.self_work = true`.

## Входные данные
- Основной JSON: `../для_теста_существенные_данные.json`.
- Markdown-референсы: пути из `lesson.materials_md.*` внутри JSON; базовая папка референсов: `../референсы`.
- Источники генерации: только основной JSON и Markdown-файлы из `lesson.materials_md.*`.

## Prompt
```text
Ты — методист дополнительного образования с экспертизой в проекте «Код будущего» и Python базового уровня.

Вход: JSON `../для_теста_существенные_данные.json` и Markdown-референсы, указанные в полях `materials_md` выбранного занятия/модуля.

Задача: Сгенерируй самостоятельную работу: 8 новых задач по аналогии с практикой темы и 10 вопросов самоконтроля. Ключи и настройки автопроверки создаются во внутреннем artifact-слое, но не показываются в ученическом HTML.

Правила:
1. Бери содержание, объём, тип занятия, уровни L1/L2/L3, цели, задачи и материалы только из JSON и Markdown-референсов.
2. Не добавляй служебные координаты, ID, SHA, leakage, candidate_internal в ученические и методические материалы.
3. Не выдумывай требования, номера пунктов, возможности платформы или шаблоны вопросов.
4. Сохраняй границы артефактов: ученический материал, МР, QA, текущий контроль и аттестация не смешиваются.
5. Объём самостоятельной работы должен быть достаточен для `lesson.hours.self_study` и `lesson.hours.raw`: 8 задач, 10 вопросов самоконтроля, порядок выполнения и требования к результату должны соответствовать указанному времени; если `lesson.hours.self_study = 0`, самостоятельную работу не генерируй как отдельный материал.
6. До ответа проверь соответствие этому skill и сохранённой структуре требований ниже.
```

## Skill
### v34 override: самостоятельная

Эти правила имеют приоритет над старыми формулировками ниже:

- Самостоятельная содержит 8 task-like практических элементов и 10 вопросов самоконтроля, если JSON явно не требует другой состав.
- Раздел краткой теории не создаётся.
- Задачи строятся по аналогии с практикой темы, но с новыми условиями, значениями, сценариями и кодом.
- Learner-facing HTML не показывает ключи, правильные ответы, заполненные шаблоны, answer flags, `correct_answers`, `autocheck_config` или internal explanations.
- Ключи самоконтроля и параметры автопроверки хранятся во внутреннем `self_work_autocheck` artifact и доступны QA/validator, но не ученику.

### JSON selectors
- `lesson.content.*` - тема и состав закрепления.
- `lesson.hours.self_study` и `lesson.hours.raw` - объём самостоятельной работы.
- `lesson.practice_tasks.*` - типы задач, по аналогии с которыми строится самостоятельная.
- `lesson.materials_md.requirements` - требования к самостоятельной.
- `lesson.materials_md.reference_examples` - эталон оформления.
- `lesson.materials_md.template_descriptions` - шаблоны вопросов самоконтроля.

### Workflow
- Не копируй практику один-в-один: условия должны быть новыми, но типы задач теми же.
- Спланируй объём по `lesson.hours.self_study` и `lesson.hours.raw`: задания, самоконтроль и требования должны покрывать всё указанное время.
- Не добавляй раздел краткой теории для подготовки.
- Сделай 8 задач, покрывающих ключевые умения темы.
- Сделай 10 вопросов самоконтроля разными шаблонами: single choice, multiple choice, 6A, 6D, 6G, 8D, 10D.
- Ключи самоконтроля включи во внутренний artifact автопроверки; в ученическом HTML ключи не показывай. Ключи к практическим задачам держи в QA, если нужен отдельный QA-артефакт.

### Expected outputs
- Самостоятельная работа: шапка, тема, цели и задачи, порядок выполнения, 8 задач, 10 вопросов самоконтроля без видимых ключей, требования к результату, источники. Ключи находятся во внутреннем artifact-слое.

### Quality gate
- Все обязательные секции из сохранённой структуры требований присутствуют или явно отмечены как неприменимые.
- Все ссылки на источники ведут на Markdown-файлы из JSON, а не на исходные `.docx`, `.pdf`, `.html`.
- Объём, тип задач, L1/L2 и ограничения L3 соответствуют JSON.
- Самостоятельная работа по количеству задач, вопросов самоконтроля и требованиям к результату достаточна для `lesson.hours.self_study` и `lesson.hours.raw`.
- Ключи и пояснения размещены только в разрешённых артефактах.
- Формулировки понятны целевой аудитории и учителю-неэксперту.

## Сохранённая структура требований
### САМОСТОЯТЕЛЬНАЯ — алгоритм и структура

### Алгоритм генерации

- **Шаг 1**: Требования по самостоятельной бери из Markdown-файлов `lesson.materials_md.requirements`.
- **Шаг 2**: Объём: 8 задач по аналогии с задачами темы + 10 вопросов самоконтроля; время бери из `lesson.hours.self_study` и `lesson.hours.raw`, модульный лимит сверяй по `module.totals`, если он задан.
- **Шаг 3**: Опора на теорию + практику.
- **Шаг 4**: Тип задач — по аналогии с задачами темы (как практика темы), не «найди ошибку».
- **Шаг 5**: Самоконтроль: 10 вопросов по шаблонам; ключи и настройки автопроверки — во внутреннем artifact-слое.
- **Шаг 6**: Оформи по Markdown-референсам из `lesson.materials_md.reference_examples`.
- **Выход**: Задания по аналогии с задачами темы + самоконтроль; ключи сохраняются внутренне и не показываются ученику.
### Структура документа (референс Самостоятельная_2.1)

- **1. Шапка УМК**: Курс → Модуль → Тема → Вид занятия (СРС) №.
- **2. Тема**: Формулировка.
- **3. Цели и задачи**: Образовательная, развивающая, практическая. Операционально.
- **4. Порядок выполнения**: 8 задач по аналогии с задачами темы.
- **5. Самоконтроль**: 10 вопросов, шаблоны из `lesson.materials_md.template_descriptions` (выбор одного ответа, выбор нескольких ответов, 6A, 6D, 6G, 8D соединение, 10D подчёркивание). Ключи — только во внутреннем artifact-слое для системы/QA, не в ученическом HTML. Код шаблона прописывается в наименование (Задача №5 6G)
- **6. Требования к результату**: Структура и объём; отчёт ученика — TNR 14, 1,5 интервала.
- **7. Источники**: Документация, статьи, видео.
Раздел «Краткая теория для подготовки» в самостоятельную НЕ включается.

ТЗ: самостоятельная работа (базовый)

Задаёт, какой должна быть самостоятельная, чтобы ученик закреплял без поддержки учителя и сам себя проверял.

- **1. Главный принцип**: Закрепление пройденного без подсказок учителя: ученик самостоятельно решает задачи ПО АНАЛОГИИ с задачами темы и сам себя проверяет (автопроверка самоконтроля). Провал: задачи дублируют практику один-в-один или самоконтроль не даёт обратной связи.
### 2. Обязательная структура

- **8 задач по аналогии с темой**: Того же типа, что задачи темы (практика), новые условия; на пройденное; разнообразные.
- **10 вопросов самоконтроля**: Шаблоны (выбор одного ответа, выбор нескольких ответов, 6A, 6D, 6G, 8D, 10D), ключи — во внутреннем artifact-слое автопроверки. Код шаблона прописывается в наименование (Задача №5 6G)
- **Без краткой теории**: Раздел теории для подготовки не включается.
- **3. Как писать**: Доступные формулировки. Задачи покрывают ключевые умения темы. Вопросы — разные шаблоны, не только выбор. Ключи для автопроверки не показываются ученику.
- **4. Требования**: Разнообразие классов ошибок; покрытие умений темы; внутренние ключи ко всем вопросам.
- **Плохо**: Хорошо
- **Все 8 задач — копии практики один-в-один; самоконтроль только из выбора ответа или без internal keys.**: Задачи по аналогии с темой (новые условия того же типа); самоконтроль из 6 типов шаблонов с внутренними ключами и автопроверкой.
