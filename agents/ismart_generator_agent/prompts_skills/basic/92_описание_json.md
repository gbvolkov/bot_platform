# Описание JSON для генератора УМК

Файл данных: `../для_теста_существенные_данные.json`

Назначение JSON: содержательное представление данных для генерации УМК. JSON не описывает табличный лист и не требует внешних табличных источников для генерации. Он содержит курс, модули, занятия, содержательные поля, уровни сложности, флаги артефактов и ссылки на Markdown-аналоги исходных файлов.

## v34 invariant

Структура входного JSON не меняется. v34 меняет правила интерпретации и внутренние artifacts генератора, но не добавляет новых обязательных входных полей.

Ключевое уточнение: `lesson.practice_tasks.*` задаёт паттерн задачи: id/номер, уровень, тип действия, проверяемый навык и ограничения. Это не обязательно финальная learner-facing формулировка. Если JSON явно не требует дословного значения, генератор создаёт новый вариант того же паттерна с другими сценариями, значениями, именами переменных, входами/выходами и кодом.

## Top-level schema
```text
course
markdown_references_base
modules[]
```

## `course`
| Поле | Значение |
|---|---|
| `title` | Название/шапка курса. |
| `norms` | Общие нормативы курса. |
| `legend` | Легенда типов занятий и обозначений. |

## `modules[]`
| Поле | Значение |
|---|---|
| `title` | Название модуля и суммарные показатели. |
| `hours` | Часы модуля, если извлечены. |
| `l1`, `l2` | Суммарные показатели уровня по модулю. |
| `lessons[]` | Занятия модуля. |
| `totals` | Итоговая строка модуля, если есть. |

## `lessons[]`
| Поле | Значение |
|---|---|
| `lesson_number` | Номер занятия из трекера как смысловой идентификатор. |
| `module` | Название модуля занятия. |
| `topic` | Тема/подтема. |
| `title`, `title_raw` | Название занятия без/с исходным маркером типа. |
| `type` | Тип занятия: теория, практика, ТП, самостоятельная, контроль и т.п. |
| `hours` | Разложение часов: `theory`, `practice`, `self_study`, `assessment`, плюс `raw`. |
| `content.general` | Общее содержание занятия. |
| `content.for_grades_8_9` | Специфика для 8-9 классов. |
| `practice_tasks.l1`, `practice_tasks.l2` | Авторитетные паттерны задач по уровням: id/номер, тип, навык и ограничения. Не обязательно финальный текст learner-facing задания. |
| `teacher_materials` | Сигналы по МР-теория/МР-практика. |
| `content_flags` | Какие артефакты нужны: теория, практика, шаблон, самостоятельная, текущий контроль, аттестация и т.д. |
| `difficulty` | Количество/проценты L1/L2, статус L3 и нарушение. |
| `materials_md` | Ссылки на Markdown-аналоги требований, референсов, целей, доноров и шаблонов. |

## `materials_md`
| Поле | Что подключать |
|---|---|
| `requirements` | Markdown-файлы требований по типу контента. |
| `reference_examples` | Markdown-референсы структуры/оформления. |
| `goals_and_tasks` | Цели, задачи и требования к составу. |
| `donor_materials` | Донорские материалы и фактура. |
| `template_descriptions` | Описания шаблонов вопросов. |

## Selector examples
```text
modules[0].lessons[0].title
modules[0].lessons[0].content.general
modules[0].lessons[0].practice_tasks.l1
modules[0].lessons[0].materials_md.requirements
modules[0].lessons[0].materials_md.reference_examples
```

## Правила использования JSON
- JSON является основным источником генерации УМК.
- Если нужно содержание темы, бери `content.general` + `content.for_grades_8_9`.
- Если нужно количество, id/order и тип задач, бери `practice_tasks` и `difficulty`, а не придумывай уровни. Конкретный вариант задания генерируй как новый вариант того же паттерна.
- Если нужен внешний документ, бери путь из `materials_md`; там уже стоят Markdown-аналоги.
- Если поле отсутствует или пустое, не заполняй его домыслами: отметь как недостающий вход или используй разрешённый общий норматив из skill-файлов.
