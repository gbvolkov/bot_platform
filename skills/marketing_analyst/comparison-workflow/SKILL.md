# Comparison Workflow

Use this workflow when the manager asks for internal comparison logic.

Preferred steps:
1. Search for each compared direction with `search_marketing_materials`.
2. Read only the strongest candidate materials with `read_marketing_material`.
3. Separate narrative claims from exact factual claims.
4. Mark numeric or specification-like claims for BI validation.
5. Mark fresh public competitor claims for web validation.

Comparison output should separate:
- fit and audience
- main value proposition
- proof or evidence candidate
- candidate-specific caveats and validation requirements
